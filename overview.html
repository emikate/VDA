Data Overview:

The dataset used for this analysis contains information about Spotify songs.

	
	Columns:
	track_id- char, unique song id
	track_name- char, song name	
	track_artist- char, artist name	
	track_popularity- double, higher the number, higher the popularity
	track_album_id- char, unique album id	
	track_album_name- char, album name	
	track_album_release_date- char, date of release	
	playlist_name- char, name of playlist	
	playlist_id- char, unique playlist id	
	playlist_genre- char, playlist genre	
	playlist_subgenre- char, playlist subgenre	
	danceability- double, higher the value, more 'danceable' song is
	energy- double, higher the value, more energy the songs contains
	key- double, estimated overall key of the track, -1 if no key detected  
	loudness- double, overall loudness of track in decibels	
	mode- double, indicates the modality: major or minor	
	speechiness- double, measured presence of spoken word 
	acousticness- double, confidence measure of whether or not track is acoustic
	instrumentalness- confidence measure of whether or not song contains vocals	
	liveness- double, detects presence of audience in song	
	valence- double, positiveness conveyed in song
	tempo- double, estimated tempo in BPM	
	duration_ms- double, duration of song in milliseconds
	
	Source: 
	Spotify's API

Goal:

The goal of this analysis is to explore the data and find meaningful correlations. Additionally, exploring the power of certain models in predicting a song's populartity using audio feautures.
